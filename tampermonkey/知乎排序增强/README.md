# 知乎排序增强 (v2.0)



这是一个为知乎（`zhihu.com`）设计的油猴（Userscript）脚本，旨在通过非侵入式的方式，一键获取当前页面内容的赞同数排序结果，极大地提升您在信息流中筛选高价值内容的效率。



## 功能亮点



本脚本的核心理念是**“读取而不干扰”**，将排序结果呈现在一个独立的辅助窗口中，完美规避了知乎复杂的前端渲染机制。

- 🖱️ 按需加载，一键触发：

  脚本在页面左下角生成一个简洁的【排序】按钮。只有当您需要时才点击触发，脚本会即时开始工作，平时不产生任何额外消耗。

- 💡 智能弹窗，结果清晰：

  点击排序后，脚本不会扰乱您当前的页面布局。所有排序结果会以列表形式，清晰地呈现在一个弹出的辅助窗口中，一目了然。

- 🔗 双重定位，快速访问：

  在弹窗的每一个结果条目后，都提供了 【新窗口打开】 和 【滚动到原文】 两个按钮，让您可以选择在新标签页中查看内容，或立即在原页面中定位到该内容，体验流畅。

- 🌐 全局搜索，杜绝遗漏：

  采用全局搜索策略，直接在整个页面中查找所有符合条件的内容项。无论知乎的页面结构如何动态变化，都能确保最大范围地捕获所有已加载的内容，避免了因容器识别失败而导致的排序不全或失败的问题。

- 🛡️ 智能过滤，结果纯净：

  脚本在解析时会自动检测并过滤掉广告、相关搜索推荐以及其他没有“赞同”按钮的非内容模块，确保最终的排序列表只包含有价值的、可排序的有效项目。

- ⚙️ 页面通用，广泛支持：

  完美支持知乎最常用的三大浏览页面：首页推荐流、问题回答页 以及 搜索结果页。

- 💪 鲁棒加载，稳定可靠：

  内置了先进的MutationObserver（DOM变更观察器），能够智能等待页面核心内容加载完成后才激活功能，彻底解决了因网络延迟或页面异步加载导致的脚本失效问题。



## 安装步骤



1. 安装脚本管理器：

   您需要在浏览器中安装一个用户脚本管理器扩展。强烈推荐使用 Tampermonkey。

   - [Chrome / Edge 版本](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)
   - [Firefox 版本](https://addons.mozilla.org/firefox/addon/tampermonkey/)

2. **安装本脚本**：

   - 点击浏览器右上角的 Tampermonkey 图标，选择“管理面板”。
   - 在打开的页面中，点击“+”号标签页来创建一个新脚本。
   - 将 **“知乎排序增强”v2.0脚本** 的完整代码复制并粘贴到编辑器中，替换掉所有模板代码。
   - 点击菜单栏的“文件” -> “保存”。



## 使用方法



1. 访问知乎网站的**首页**、**搜索结果页**或**问题回答页**。
2. 页面加载完成后，您会在**左下角**看到一个蓝色的 **【排序】** 按钮。
3. **（重要）** 如果您想排序更多内容，请先**向下滚动页面**，让知乎加载出更多的信息流。
4. 点击【排序】按钮，脚本会开始抓取和处理数据。
5. 处理完成后，屏幕中央会弹出一个包含排序结果的窗口。
6. 您可以在弹窗中浏览结果，并通过点击按钮来访问或定位原文。点击弹窗外部的灰色区域或右上角的 "×" 即可关闭。

# **技术交接文档**



项目版本: 2.0

核心目标: 在zhihu.com网站上，以非侵入式的弹窗形式，为用户提供对当前已加载内容的赞同数排序功能，并确保脚本在知乎动态、复杂的React前端环境下长期稳定运行。



#### **1. 核心架构设计**



本脚本的架构核心是**“侦察-抓取-呈现”三层分离模式**，旨在完全规避对知乎主页面DOM的直接修改，从而绕开其虚拟DOM（Virtual DOM）的覆盖和重置机制。

- **侦察层 (Observer-Driven Initialization)**：
  - **实现**: `initialize`函数是脚本的入口，其核心是一个`MutationObserver`实例。
  - **策略**: 摒弃了不可靠的`setTimeout`延时等待。该观察器会监视整个`document.body`的动态变化，等待配置中定义的**第一个内容项**（`itemSelector`）被渲染到DOM中。
  - **优势**: 确保了只有在页面核心内容开始渲染后，脚本的UI（排序按钮）才会被创建和注入，从根本上解决了时机问题。一旦按钮成功创建，观察器会立即`disconnect()`，以节省性能。
- **抓取层 (Global Search & Attribute-Based Selection)**：
  - **全局搜索**: `processAndShowSortedList`函数执行时，它直接调用`document.querySelectorAll(config.itemSelector)`在**整个文档范围**内进行搜索。这放弃了寻找特定“父容器”并在其内部搜索的传统模式，以应对知乎DOM结构不稳定的问题。
  - **属性选择器**: 赞同按钮的定位是本脚本最关键的技术突破。`voteSelector`被硬编码为`button[aria-label^="赞同"]`。这使得脚本不再依赖于知乎随时可能变更的CSS类名，而是依赖于为辅助功能（Accessibility）设计的、语义化且相对稳定的`aria-label`属性。
  - **过滤机制**: 在遍历所有找到的内容项时，通过检查`item.querySelector(config.voteSelector)`是否存在，隐式地过滤掉了广告、推荐等不包含赞同按钮的非内容模块。
- **呈现层 (Decoupled Modal UI)**：
  - **实现**: `displayResultsInModal`函数负责所有UI的渲染。它通过`GM_addStyle`注入样式，并通过纯JavaScript动态创建弹窗的HTML结构。
  - **解耦**: 这个弹窗是完全独立的DOM结构，被直接挂载在`document.body`下。它与知乎页面的唯一交互是通过`scrollIntoView`方法实现的“滚动到原文”功能，这是一种良性的、非破坏性的交互。



#### **2. 关键函数详解**



- `initialize()`: **脚本启动器与侦察兵**。负责识别页面类型，并部署`MutationObserver`来监视DOM，直到第一个目标内容项出现，然后调用`createFixedButton`。
- `createFixedButton()`: **UI构造器**。负责创建左下角的全局【排序】按钮，并为其绑定点击事件。
- `processAndShowSortedList()`: **数据处理器**。是用户点击按钮后的核心工作流。
  1. 执行**全局搜索**获取所有内容项。
  2. 遍历内容项，执行**过滤**（检查赞同按钮是否存在）。
  3. 对有效项进行**数据提取**（赞同数、标题、链接）。
  4. 执行**排序**。
  5. 调用`displayResultsInModal`将结果移交呈现层。
- `displayResultsInModal()`: **UI渲染器**。接收排序好的数据，动态构建HTML弹窗，将其注入页面，并为弹窗内的所有交互元素（关闭、新窗口打开、滚动到原文）绑定事件监听器。
- `parseVoteCount()`: **工具函数**。一个健壮的解析器，负责将“1.2 万”、“2,333”等格式不一的赞同数字符串精确转换为数字。



#### **3. 数据流**



1. **页面加载** -> `initialize()`启动`MutationObserver`。
2. **知乎JS渲染内容** -> `Observer`检测到第一个`itemSelector`出现。
3. -> `createFixedButton()`创建【排序】按钮 -> `Observer`停止工作。
4. **用户滚动加载更多内容** (可选)。
5. **用户点击【排序】** -> `processAndShowSortedList()`被调用。
6. -> **全局搜索**`document.querySelectorAll()`获取所有`items`。
7. -> **遍历与过滤**: `items.forEach`循环，通过`item.querySelector(voteSelector)`检查并筛选出有效项目。
8. -> **数据提取**: 从有效项目中解析出`votes`, `title`, `url`等信息。
9. -> `itemsData.sort()`执行内部排序。
10. -> `displayResultsInModal()`接收排序后的数据，创建并显示弹窗。



#### **4. 未来维护与扩展指南**



- **网站布局变更**:
  - **最可能失效点**: `pageConfigs`对象中的选择器是维护的重点。
  - **`itemSelector`失效** (e.g., `.TopstoryItem`变更): 如果脚本完全找不到内容（结果为0），应优先检查该页面对应的`itemSelector`是否已过时。
  - **`voteSelector`失效**: 如果脚本能找到内容项，但解析出的有效项目为0，说明赞同按钮的`aria-label`属性可能发生了变化。请检查`voteSelector`变量。
  - **`titleSelector`失效**: 如果弹窗结果中标题显示不正确，请检查对应页面的`titleSelector`。
- **新增功能**:
  - **支持新页面**: 在`detectPageType`中增加新的URL判断逻辑，并在`pageConfigs`中添加新页面的`itemSelector`等配置即可，无需改动核心流程。
- **调试**:
  - 本脚本已包含清晰的`console.log`日志输出。进行调试时，请按F12打开开发者工具，在“控制台”(Console)中即可追踪脚本的每一步执行状态。
